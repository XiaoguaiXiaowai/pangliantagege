# 个人网站开发计划文档

## 一、项目背景与目标

本项目旨在开发一个 **个人展示 + 互动 + 创作输出** 的综合型个人网站，用于：
- 系统化展示个人简历与能力
- 与访客进行低门槛互动（留言 / AI 助理）
- 长期沉淀个人作品（工具 / 音乐 / 视频）

整体目标：**结构清晰、模块解耦、可长期迭代、成本可控**。

---

## 二、网站整体结构设计

### 2.1 页面结构形式

- 采用 **统一 Header + Footer 的单页应用（SPA）结构**
- Header：
  - 横向排列多个 Tab
  - 点击 Tab 后，主体区域切换至对应模块
- Footer：
  - 版权信息
  - 联系方式 / 备案信息（如需要）

> 前端整体由 Vue Router 控制路由切换，保证无刷新体验。

### 2.2 主导航 Tab 规划

| Tab 名称 | 对应模块 |
|--------|----------|
| 关于我 | 个人简历模块 |
| 留言板 | 留言互动模块 |
| AI 助理 | AI 助理模块 |
| 小工具 | 小工具分享模块 |
| 才艺 | 上点儿才艺模块 |

---

## 三、功能模块详细设计

### 3.1 个人简历模块

**功能定位：** 结构化展示个人经历，兼顾阅读体验与快速定位。

#### 3.1.1 功能说明

- 页面顶部：
  - 提供 **PDF 版简历下载链接**
- 页面主体：
  - 纵向滚动的简历内容
  - 包含但不限于：
    - 基本信息
    - 技能栈
    - 项目经历
    - 工作经历
    - 教育背景

#### 3.1.2 侧边栏设计

- 页面左侧固定侧边栏
- 自动同步显示各内容块的 Title
- 点击侧边栏标题：
  - 页面滚动到对应内容区域

> 技术实现：Intersection Observer + 锚点滚动

---

### 3.2 留言板模块

**功能定位：** 提供低门槛访客互动渠道。

#### 3.2.1 留言功能

- 无需注册、无需登录
- 留言时可填写：
  - 昵称（可选）
  - 留言内容（必填）
- 默认昵称规则：
  - `神秘人 + 自增编号`

#### 3.2.2 回复功能

- 站长可直接在页面内回复留言
- 回复以「站长回复」形式展示
- 支持多条留言、多条回复

#### 3.2.3 基本风控

- 留言频率限制（IP / 时间窗口）
- 简单敏感词过滤

---

### 3.3 AI 助理模块

**功能定位：** 个人网站的智能互动入口（默认关闭）。

#### 3.3.1 AI 使用状态设计

- 默认状态：**不可用**
- 不可用时统一回复：

> “主人穷，还没给 API 充值，我去叫他哈”

#### 3.3.2 AI 功能开启后

- 接入第三方 AI API（如 OpenAI / 国内大模型）
- 支持基础问答对话

#### 3.3.3 使用通知机制

- 当有访客尝试使用 AI 助理时：
  - 触发短信通知站长
  - 内容示例：
    - “有人正在访问你的 AI 助理模块”

> 短信服务可通过第三方短信平台实现

---

### 3.4 小工具分享模块

**功能定位：** 展示并分享自制的小工具。

#### 3.4.1 功能设计

- 工具列表展示：
  - 工具名称
  - 简介说明
  - 使用说明
- 支持：
  - 外链跳转
  - 页面内工具（iframe 或前端实现）

#### 3.4.2 后期扩展

- 工具分类
- 热度 / 使用次数统计

---

### 3.5 上点儿才艺模块

**功能定位：** 展示个人创作内容，增强个人特色。

#### 3.5.1 内容形式

- 演出视频
- 原创音乐
- 其他表演类内容

#### 3.5.2 展示方式

- 视频 / 音频播放器嵌入
- 支持平台：
  - 本地上传
  - 第三方平台嵌入（如 Bilibili / YouTube）

---

## 四、技术架构设计

### 4.1 架构模式

- **前后端分离架构**
- 前端负责展示与交互
- 后端提供 RESTful API

---

### 4.2 前端技术栈

- 框架：Vue 3
- 路由管理：Vue Router
- 状态管理：Pinia
- UI：自定义样式 / 轻量 UI 库
- 构建工具：Vite

---

### 4.3 后端技术栈

- 框架：Django
- API：Django REST Framework
- 权限：Token / 简单管理认证
- 定时任务：Celery（可选）

---

### 4.4 数据库设计

- 数据库：MySQL
- 核心表：
  - 用户（站长）表
  - 留言表
  - 留言回复表
  - 工具信息表
  - 才艺内容表

---

## 五、部署与运维规划

- 前端：Nginx + 静态资源部署
- 后端：Gunicorn + Django
- 数据库：MySQL
- HTTPS：Let's Encrypt
- 日志：
  - 访问日志
  - 错误日志

---

## 六、迭代建议（阶段性）

1. 第一阶段：
   - 网站骨架 + 简历模块
2. 第二阶段：
   - 留言板模块
3. 第三阶段：
   - AI 助理模块（默认关闭）
4. 第四阶段：
   - 小工具 & 才艺模块

---

## 七、总结

该个人网站以 **展示 + 互动 + 创作** 为核心，整体设计偏长期可维护与持续扩展，适合逐步打磨，最终形成具有个人风格和技术含量的线上名片。
